# 功能状态文档

## 文档目的

这份文档用于跟踪 `myagent` 作为“本地优先代码仓库智能助手”时，对外可宣称的产品功能状态。

关注点：

- 面向用户的能力
- 演示场景
- 简历可写的亮点
- 当前已完成 / 未完成功能

## 目标产品方向

建议当前把项目产品方向聚焦为：

**本地优先的代码仓库智能助手**

核心问题：

- 让模型安全地理解本地代码仓库
- 让模型能通过工具真实检查工作区，而不是只做文本猜测
- 让上下文、偏好和任务事实可持续累积
- 让工具扩展能力通过 MCP 接入

## 目标用户

### 1. 个人开发者

需要：

- 看项目结构
- 理解代码
- 解释脚本与命令输出
- 在本地上下文里持续对话

### 2. 小团队 / 实验室

需要：

- 一个可控的本地 Agent 内核
- 统一接内部脚本或 MCP 工具
- 后续扩展为内部助手

## 当前可用功能

## 业务能力

从“本地优先代码仓库智能助手”的角度，当前项目可以逐步演进为以下业务能力。

### 1. 仓库理解与问答

状态：**基础可用**

目标能力：

- 解释项目目录结构
- 总结 README、配置文件和脚本
- 帮用户快速理解陌生仓库
- 定位入口文件、核心模块、测试文件

当前基础：

- 已支持 `list_dir`
- 已支持 `repo_search`
- 已支持 `read_file`
- 已支持多轮 tool loop

后续增强方向：

- 更强的仓库级摘要
- 跨文件关系分析
- 更稳定的入口定位

### 2. 代码解释与开发辅助

状态：**基础可用**

目标能力：

- 解释类、函数、模块职责
- 总结某段代码在整体项目中的作用
- 帮助理解旧代码或陌生代码
- 输出开发建议和下一步排查方向

当前基础：

- 已支持文件读取
- 已支持 session 上下文
- 已支持 memory v2

后续增强方向：

- 更强的代码关系追踪
- 针对仓库任务的专用 prompt

### 3. 本地调试与命令辅助

状态：**已具备基础能力**

目标能力：

- 执行受控命令并解释输出
- 帮用户检查环境、版本、脚本结果
- 分析测试或命令失败原因
- 给出下一步调试建议

当前基础：

- 已支持 `run_command`
- 已有命令安全策略
- 已有 tool result 结构化返回

后续增强方向：

- 更精细的命令分级
- 更清晰的失败原因归类
- 更强的调试报告输出

### 4. 会话化开发助手

状态：**已具备基础能力**

目标能力：

- 记住当前正在处理的问题
- 记住用户偏好、仓库事实和近期任务
- 在后续对话中延续上下文
- 减少反复说明背景的成本

当前基础：

- session persistence
- memory v2
- 摘要、事实、task memory 与最小长期记忆动态注入

后续增强方向：

- 更明确的任务记忆
- 更完整的长期检索
- 更好的相关性检索
- 更强的摘要压缩与 consolidation

### 5. MCP 工具扩展工作台

状态：**最小版已具备**

目标能力：

- 通过 MCP 扩展外部工具能力
- 让 agent 不只操作本地文件和命令
- 后续接入更多内部或外部工作流工具

当前基础：

- 多个 `stdio` MCP server
- `tools/list`
- `tools/call`
- MCP tools 自动包装进现有工具系统

后续增强方向：

- 更清晰的命名空间
- 更完整的 MCP 协议支持

### 6. 服务化访问入口

状态：**最小版已具备**

目标能力：

- 通过 HTTP 访问 agent，而不只依赖 CLI
- 支持 session 化对话
- 让后续前端、channel、自动化系统接入更容易

当前基础：

- `GET /health`
- `POST /chat`
- `GET /sessions/{id}`

后续增强方向：

- streaming 输出
- 更完整的请求参数
- 鉴权与限流

### 代码仓库理解

状态：**已实现**

可做：

- 列出当前目录并解释项目结构
- 搜索候选文件并定位可能相关实现
- 读取文件并总结内容
- 在当前工作区内做基础代码理解

依赖能力：

- `list_dir`
- `repo_search`
- `read_file`
- 多轮 agent loop

### 本地命令辅助

状态：**已实现**

可做：

- 执行受限的只读命令
- 解释命令输出
- 在代码仓库任务里辅助定位问题

依赖能力：

- `run_command`
- tool safety policy

### 持续会话与基础记忆

状态：**已实现**

可做：

- 保留 session
- 记住近期对话摘要
- 记住简单事实
- 记住当前任务、已完成步骤、待办与阻塞
- 在不同 session 之间复用项目级长期事实与任务结论
- 在后续对话中复用这些信息

依赖能力：

- session persistence
- memory v2

### MCP 工具扩展

状态：**已实现（最小版）**

可做：

- 接入多个 `stdio` MCP server
- 自动发现 MCP tools
- 让模型直接调用 MCP 暴露的工具

当前限制：

- 只支持工具能力

### 可追踪调试

状态：**已实现**

可做：

- 查看 session 级日志
- 查看 tool/provider 执行元数据
- 查看 provider 原始响应

## 当前还不能对外宣称的功能

这些能力现在还不应该写成“已完成”：

- Web 页面
- 多 channel 接入
- 更强的 skill 资源加载与治理
- 后台任务 / 定时任务 / 主动提醒
- 复杂长期记忆检索
- 多 agent 协作

## 简历可写亮点

当前已经可以写的技术点：

- 本地优先 Agent Runtime
- OpenAI-compatible provider abstraction
- Structured tool execution and safety policy
- Session persistence, task memory, and minimal long-term retrieval
- `SKILL.md`-based skill runtime with local discovery
- Runtime tracing and observability
- Multi-server MCP integration
- Minimal HTTP API service layer

## 推荐演示场景

适合做演示的视频或 README demo：

1. “读取仓库结构并总结项目”
2. “搜索某个函数/模块在哪里实现，并解释实现”
3. “执行 `python --version` 并解释结果”
4. “记住用户身份/项目事实，并在后续 session 中复用”
5. “调用 MCP 工具完成额外任务”
6. “通过 HTTP API 发起对话并查看 session”
